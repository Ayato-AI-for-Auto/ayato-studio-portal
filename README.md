# Ayato Studio Portal (Intelligence Synergy)

AIエージェントによる自動分析（フロー型）と人間による深い洞察（ストック型）を融合させた、次世代型インテリジェンス・メディア。
情報の「鮮度」と「権威性」を同時に提供する三層構造のコンテンツ・アーキテクチャを採用しています。

---

## Content Layers (三層構造)

| レイヤー | コンテンツ | データソース | 目的 | 速度 |
| :--- | :--- | :--- | :--- | :--- |
| **Tier 1: 速報** | 市場ニュース、技術論文 | Supabase (AI) | 最新の市場動向の把握 | 高速 |
| **Tier 2: ブログ** | 週刊レビュー、独自考察 | Supabase (AI) + Local (Human) | 戦略的な洞察と理論の提供 | 中速 |
| **Tier 3: 実績** | サービス、プロダクト紹介 | Local (Human) | 信頼構築とポートフォリオ管理 | 低速 |

---

## How to: 人間による記事の反映 (Stock Content)

人間が執筆した記事（ストック型資産）を Web ページに反映させるためのワークフローです。
**データベース操作は不要** で、ローカルの Markdown ファイルを編集して GitHub に Push するだけで自動的に反映されます。

### 1. 記事作成の場所
以下のディレクトリに `.md` ファイルを作成します。
- **ブログ・技術論**: `src/content/blog/`
- **サービス・実績紹介**: `src/content/services/`

### 2. ファイルフォーマット (Markdown + Frontmatter)
ファイルの冒頭に YAML 形式でメタデータを記述します。

```markdown
---
title: 記事のタイトル
description: 概要（カード一覧に表示されます）
date: 2026-04-03
category: blog # または services
---

# ここから本文
Markdown 形式でグラフ、コード、箇条書きなどが利用可能です。
```

### 3. 反映プロセス
1.  ローカルで記事を執筆・保存。
2.  `git add` -> `git commit` -> `git push` を実行。
3.  Cloudflare Pages 等の CI/CD が自動的にビルドを開始し、数分でサイトに反映。

## System Strategy: 情報の「フロー」と「ストック」の使い分け

このシステムを最大限に活用するための戦略的アプローチです。

- **AI の役割 (Flow & Synthesis)**: 
    - 24/7 で世界中のノイズをスキャンし、重要な変化のみを抽出（速報レイヤー）。
    - 1週間分の情報を統合分析し、人間が深い考察を行うための「下地」を生成（週刊レビュー）。
- **人間の役割 (Stock & Brand)**: 
    - AI が提示した分析を基に、より高次な「ビジョン」や「独自の解釈」を記事として蓄積。
    - 自身の開発実績を LP レベルのクオリティで紹介し、個人のブランド価値（E-E-A-T）を確立。

**AI が「情報の速度」を、人間が「知恵の深さ」を担うことで、情報の洪水に沈まないインテリジェンス・ハブを構築します。**
